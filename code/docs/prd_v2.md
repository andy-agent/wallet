## 2. 核心业务规则

### 2.1 订单规则

- 订单有效期：**15 分钟**
- 支付确认数（可配置）：
  - SOL: **12** 个确认
  - USDT-TRC20: **19** 个确认
- 汇率锁定：订单创建时锁定，超时需重新创建
- 金额容差：**±0.1%**（考虑链上精度）
- 地址模式：**每笔订单独立收款地址**（非共享地址+memo）

### 2.2 幂等性规则（强制）

| 操作 | 幂等键 | 行为 |
|------|--------|------|
| 创建订单 | device_id + plan_id + purchase_type | 15分钟内禁止重复创建 |
| 支付确认 | order_id + tx_hash | 同一交易只处理一次 |
| 开通账号 | order_id | 同一订单只开通一次（数据库唯一约束） |
| 续费处理 | order_id | 同一订单只续费一次 |
| 订阅查询 | client_token | 正常查询 |

### 2.3 安全规则

- 客户端**仅允许**访问 `/client/v1/*` 接口
- 管理接口需 Admin Token（客户端不可访问）
- 私钥服务端加密存储（AES-256-GCM）
- 每笔订单使用独立收款地址
- JWT Token 短期有效（建议 30 天）

## 3. 订单状态机（10 个状态）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           订单状态机                                     │
└─────────────────────────────────────────────────────────────────────────┘

pending_payment ──→ seen_onchain ──→ confirming ──→ paid_success ──→ fulfilled
       │                  │              │              │
       │                  │              │              └─ amount mismatch
       │                  │              │                 → underpaid/overpaid
       │                  │              │
       │                  │              └─ confirm timeout → expired
       │                  │
       │                  └─ wrong asset/amount → failed
       │
       └─ 15min timeout ───────────────────────────────→ expired

特殊状态:
- cancelled: 人工取消
- late_paid: 过期后到账（不自动开通）
```

### 状态说明

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| `pending_payment` | 待支付 | 订单创建 |
| `seen_onchain` | 已发现交易 | 检测到链上交易 |
| `confirming` | 确认中 | 交易已确认但不足最小确认数 |
| `paid_success` | 已确认且金额匹配 | 达到最小确认数，金额正确 |
| `fulfilled` | 账号已开通 | 成功调用 Marzban 创建/续费用户 |
| `expired` | 超时 | 超过 15 分钟未支付 |
| `underpaid` | 少付 | 到账金额 < 订单金额 - 容差 |
| `overpaid` | 多付 | 到账金额 > 订单金额 + 容差 |
| `failed` | 支付校验失败 | 错误币种、错误地址等 |
| `late_paid` | 过期后到账 | 订单过期后检测到支付 |

## 4. 新购 vs 续费

### 4.1 新购（purchase_type = new）

1. 用户选择套餐，创建订单
2. 完成支付，订单状态 → paid_success
3. 系统自动：
   - 生成唯一用户名（marzban_username）
   - 调用 Marzban 创建用户
   - 设置流量和到期时间
   - 创建 client_session（access_token + refresh_token）
   - 订单状态 → fulfilled
4. 客户端收到 token，拉取订阅

### 4.2 续费（purchase_type = renew）

1. 已登录用户（带 client_token）选择套餐
2. 创建订单时传入 purchase_type = renew
3. 完成支付，订单状态 → paid_success
4. 系统自动：
   - 根据 client_token 找到原用户
   - 调用 Marzban 更新用户（延长到期时间或叠加流量）
   - 订单状态 → fulfilled
5. 客户端刷新订阅，显示新到期时间

### 4.3 幂等保障

```sql
-- 开通账号幂等约束
CREATE UNIQUE INDEX idx_orders_fulfill ON orders(id) 
WHERE status = 'fulfilled';

-- 同一交易只能绑定一个订单
CREATE UNIQUE INDEX idx_orders_tx_hash ON orders(tx_hash) 
WHERE tx_hash IS NOT NULL;
```

## 5. 异常处理规则

| 场景 | 处理规则 |
|------|----------|
| 少付 (underpaid) | 不自动开通，进入异常单，后台人工处理 |
| 多付 (overpaid) | 正常开通，多付部分计入异常资金池 |
| 误付币种 | 不自动开通，无法识别时进入 failed |
| 重复支付（同一订单） | 第一笔成功，其余进入 late_paid 或异常池 |
| 过期后到账 (late_paid) | 不自动开通，后台人工补单或退款 |
| 链上回滚 | 确认数回退时重新计算，可能回到 confirming |

## 6. 客户端流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ 启动App  │────→│ 获取套餐 │────→│ 选择套餐 │────→│ 选择支付 │
└─────────┘     └─────────┘     └─────────┘     └────┬────┘
                                                      │
┌─────────┐     ┌─────────┐     ┌─────────┐          │
│ 自动连接 │←────│ 导入节点 │←────│ 支付成功 │←─────────┘
└─────────┘     └─────────┘     └─────────┘
                                     ▲
┌─────────┐     ┌─────────┐          │
│ 显示二维码│────→│ 用户转账 │─────────┘
└─────────┘     └─────────┘
```

### 6.1 轮询策略

```
初始延迟: 3s
退避策略: 3s → 3s → 5s → 5s → 8s → 8s → 10s (固定)
最大轮询: 订单过期或支付完成
后台轮询: 降低频率至 30s
```

## 8. 验收标准

### 8.1 功能验收

| 场景 | 验收标准 |
|------|----------|
| 创建订单 | 返回正确的收款地址、金额、二维码 |
| SOL 支付 | 测试网转账后 2 分钟内状态变为 fulfilled |
| USDT-TRC20 支付 | 测试网转账后 2 分钟内状态变为 fulfilled |
| 新购开通 | 支付成功后自动生成可用订阅 |
| 续费成功 | 到期时间或流量正确增加 |
| 幂等性 | 重复支付只开通一次，数据库无重复记录 |
| 过期处理 | 15 分钟后未支付订单自动 expired |

### 8.2 性能验收

| 指标 | 目标 |
|------|------|
| 订单创建 API | < 500ms (P95) |
| 支付状态查询 | < 200ms |
| 订阅生成 | < 1s |
| 链上检测延迟 | < 30s（检测到交易后） |

### 8.3 安全验收

| 检查项 | 标准 |
|--------|------|
| 客户端无 Admin 接口访问 | 返回 403 |
| 私钥加密存储 | 数据库中无明文私钥 |
| 订单超时自动关闭 | 15 分钟后 expired |
| 重复支付处理 | 同一 tx_hash 只处理一次 |
| Token 过期处理 | 返回 401，客户端需重新登录 |

## 9. 风险与应对

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| 链上 RPC 不稳定 | 高 | 中 | 多 RPC 备份，支持 Mock 模式 |
| 汇率波动 | 中 | 高 | 订单创建时锁定汇率 |
| 私钥泄露 | 极高 | 低 | 加密存储，地址池轮换 |
| 并发订单冲突 | 中 | 中 | 数据库唯一约束 + 幂等锁 |
| Marzban API 变更 | 高 | 低 | 抽象 Marzban Client 层 |
| 链上拥堵 | 中 | 中 | 可配置确认数，测试网预演 |
