# VPN 节点控制面人类操作流程说明

## 目标

本流程用于维护当前 VPN 节点控制面模型：

- 业务后端是唯一业务真相源
- Marzban 负责节点接入、订阅输出、用户代理能力
- App 前台只读取业务后端整理后的地区/线路/节点结果

## 当前控制面边界

- 商品/套餐、区域权限、节点选择：
  - 由 `code/backend` 负责
- 节点接入、Host Settings、用户订阅能力：
  - 由 Marzban 负责
- 当前真实出口地区：
  - 由客户端运行时校验，不作为主选择源

## 新节点接入流程

1. 在目标服务器部署 `marzban-node`
2. 在 Marzban 后台完成节点接入
3. 在 Marzban `Host Settings` 中为节点准备可下发的 host 项
4. 确认订阅输出中已经出现该 host 对应 entry
5. 在业务后端将该节点映射到对应线路

## 线路配置原则

- `region` 表示地区，如 `HK` / `JP`
- `line` 表示可售线路，如 `HK_BASIC` / `HK_PREMIUM`
- `node` 表示真实连接端点

当前实现中：

- 前端主显示读取“当前选中的线路/节点”
- `vpn/status` 返回：
  - `selectedRegionCode`
  - `selectedRegionName`
  - `selectedLineCode`
  - `selectedLineName`
  - `selectedNodeId`
  - `selectedNodeName`

## 套餐与权限

购买完成后，业务后端按套餐决定用户可访问的线路集合。

推荐规则：

- 单区套餐：
  - 只开放对应 `line`
- 多区共享套餐：
  - 开放多个 `line`
- 独享套餐：
  - 开放一个独享 `line`

当用户切到未开通线路时，前端提示：

- `当前套餐未开通该线路`

若目标线路属于更高等级套餐，可提示：

- `你当前已开通该区域基础线路，高级线路暂未开通`

## 用户切换节点流程

1. App 调用 `GET /api/client/v1/vpn/regions`
2. App 调用 `GET /api/client/v1/vpn/nodes?lineCode=...`
3. 用户在地区/节点页选择目标节点
4. App 调用 `POST /api/client/v1/vpn/selection`
5. 后端保存：
   - `selectedLineCode`
   - `selectedNodeId`
6. App 按节点 `host:port` 匹配本地订阅中的 entry，并切换本地选中服务器
7. 返回总览页时，前台状态显示后端返回的选中线路/节点信息

## 运行时校验

连接成功后，客户端仍需做运行时校验：

- 真实出口 IP
- 出口国家/地区
- 当前连接状态

如果运行时出口地区与选中线路不一致，应视为异常，但不直接覆盖用户当前选中的线路显示。

## 验收要点

1. 登录后可以正常同步订阅并导入本地节点
2. `切换节点` 页面展示的是后端真实节点目录
3. 选择节点后：
   - 后端 `vpn/status` 返回新的 `selectedLineCode / selectedNodeId`
   - App 本地选中服务器同步变化
4. 返回总览后：
   - 前台显示已选线路
   - 连接/断开状态能实时刷新
5. VPN 启动后：
   - 服务常驻
   - 流量可代理

## 当前实现限制

- 当前生产线路目录仍以现有 `vpn_regions` 数据为准
- 若未来引入更多区域/线路，业务后端需要继续补齐更明确的线路映射和运营配置
