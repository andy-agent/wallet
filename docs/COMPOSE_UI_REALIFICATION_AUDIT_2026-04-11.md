## 额外风险点

- `P0/P1/P2/P2Extended` NavGraph 的函数签名默认参数仍是 `MockP0Repository()` / `MockCryptoVpnRepository()`，见 [P0NavGraph.kt](/Users/cnyirui/git/projects/liaojiang/code/Android/V2rayNG/app/src/main/java/com/v2ray/ang/composeui/navigation/P0NavGraph.kt)、[P1NavGraph.kt](/Users/cnyirui/git/projects/liaojiang/code/Android/V2rayNG/app/src/main/java/com/v2ray/ang/composeui/navigation/P1NavGraph.kt)、[P2CoreNavGraph.kt](/Users/cnyirui/git/projects/liaojiang/code/Android/V2rayNG/app/src/main/java/com/v2ray/ang/composeui/navigation/P2CoreNavGraph.kt)、[P2ExtendedNavGraph.kt](/Users/cnyirui/git/projects/liaojiang/code/Android/V2rayNG/app/src/main/java/com/v2ray/ang/composeui/navigation/P2ExtendedNavGraph.kt)。当前生产入口会覆盖它们，但这仍是高风险默认值。
- 页面文件内大量 `@Preview` 使用 `Mock*Repository` 或 preview state，这本身不等于生产路径污染；但 `AssetDetail` 已确认把 preview state 回退进了生产仓库逻辑，是明确违规。

## 总结论

- 当前新 UI **没有任何页面达到 A 类**。
- 已真实落地的是“部分业务链路”，不是“Compose UI 页面”。
- 当前最常见的伪真实模式：
  - `Real*Repository` 内部返回本地拼装视图
  - NavGraph 用硬编码订单号/planId/txId/钱包 ID 导航
  - 页面展示部分真实字段，但动作仍是假导航
  - 页面只读本地缓存，不读真实服务端对象
  - 页面为空时保留 preview/本地空壳/模板页
  - 钱包扩展页大面积只是“本地流程说明”，不是业务完成态
