## 2. 产品背景
这是一个以 VPN 订阅为核心收入、以多链自托管钱包为关键辅助模块的 Android App。

用户主路径：
- 下载 App
- 邮箱注册 / 登录
- 选择套餐
- 创建订单
- 链上支付
- 自动开通订阅
- 选择区域
- 连接 VPN

增长路径：
- 绑定邀请码
- 产生一级 / 二级佣金
- 查看账本
- 提现到链上地址

钱包路径：
- 创建 / 导入本地钱包
- 查看资产
- 收款 / 发送
- 在订单场景中直接支付

## 3. 已冻结业务规则
AI 设计时必须遵守以下规则，不能擅自改产品逻辑：

1. 注册方式：邮箱优先，不做手机号。
2. 设备策略：单活跃 session，新设备登录会挤下旧设备。
3. VPN 协议：VLESS + Reality + XTLS/Vision。
4. 支付资产：`SOL on Solana`、`USDT on Solana`、`USDT on TRON/TRC20`。
5. 钱包签名：客户端本地签名。
6. 广播边界：客户端直连广播为默认，后端代理广播为兜底。
7. 严禁：后端代签名、后端托管用户私钥。
8. 分佣：一级 25%，二级 5%。
9. 提现：最低 10 USDT，默认 `USDT on Solana`。
10. App 分发：官网 APK 分发，不考虑应用商店首发。

## 4. 设计目标
页面设计必须同时满足以下目标：

1. 用户一眼知道这是“VPN + 钱包 + 链上支付”的产品，但视觉重心是 VPN 订阅，不是交易所。
2. 页面风格需要传达：
   - 安全感
   - 稳定感
   - 专业感
   - 轻高级感
3. 避免以下错误方向：
   - 交易所式强投机视觉
   - 赛博朋克过度霓虹
   - 泛 Web3 钱包模板风
   - 极简到信息层级不清
4. 设计必须偏移动端原生 App，而不是把网页套壳成手机页面。
