## 2. 责任划分矩阵

| 职责 | API 层 | Sol 链侧 | USDT 链侧 | 说明 |
|------|--------|----------|-----------|------|
| **地址管理** ||||
| 地址池分配/释放 | ✅ | ❌ | ❌ | API 层维护 `payment_addresses` 表状态机 |
| 地址生成 (HD) | ❌ | ✅ | ✅ | 链侧按索引派生地址，私钥不出链侧 |
| 私钥存储/加密 | ❌ | ✅ | ✅ | 链侧本地加密存储，API 层不持有 |
| **支付检测** ||||
| 扫描待支付订单 | ✅ | ❌ | ❌ | API 层 `scanner.py` 定时任务触发 |
| RPC 调用获取交易 | ❌ | ✅ | ✅ | 链侧提供 `/payment/detect` 接口 |
| 确认数计算 | ❌ | ✅ | ✅ | 链侧返回当前确认数 |
| 订单状态推进 | ✅ | ❌ | ❌ | API 层状态机 `state_machine.py` 驱动 |
| **账本管理** ||||
| 订单生命周期 | ✅ | ❌ | ❌ | `orders` 表，API 层维护 |
| 用户余额/佣金 | ✅ | ❌ | ❌ | `referral/commissions` 模块 |
| 扫账/归集记录 | ✅ | ❌ | ❌ | `sweep_records` 表 |
| **交易操作** ||||
| 扫账交易广播 | ❌ | ✅ | ✅ | 链侧签名并广播，返回 tx_hash |
| 提现交易广播 | ❌ | ✅ | ✅ | 同上 |
| 交易状态查询 | ❌ | ✅ | ✅ | 链侧提供 `/tx/{hash}` 接口 |
| **汇率/定价** ||||
| 法币-加密汇率 | ✅ | ❌ | ❌ | API 层 `fx_rate.py` 维护 |
| 金额计算 | ✅ | ❌ | ❌ | API 层按汇率计算 `amount_crypto` |

## 3. 链侧服务接口契约

### 3.1 Solana 链侧 (`sol.residential-agent.com`)

| 端点 | 方法 | 职责 | 调用方 |
|------|------|------|--------|
| `/health` | GET | 健康检查 | API Worker |
| `/address/generate` | POST | 生成 HD 地址 | API 初始化/扩容 |
| `/balance` | POST | 查询 SOL/SPL 余额 | API Scanner |
| `/payment/detect` | POST | 检测地址入账 | API Scanner |
| `/tx/broadcast` | POST | 广播扫账交易 | API Sweeper |
| `/tx/{hash}` | GET | 查询交易状态 | API Worker |

### 3.2 USDT 链侧 (`usdt.residential-agent.com`)

| 端点 | 方法 | 职责 | 调用方 |
|------|------|------|--------|
| `/health` | GET | 健康检查 | API Worker |
| `/address/generate` | POST | 生成 Tron 地址 | API 初始化/扩容 |
| `/balance` | POST | 查询 TRX/USDT 余额 | API Scanner |
| `/payment/detect` | POST | 检测 USDT 转账 | API Scanner |
| `/tx/broadcast` | POST | 广播扫账交易 | API Sweeper |
| `/tx/{hash}` | GET | 查询交易状态 | API Worker |

## 6. 回退方案

若链侧服务不可用时：

1. **API 层保留直连能力**: `integrations/solana.py` 和 `tron.py` 暂不删除
2. **配置开关**: 通过 `WALLET_SERVICE_MODE=local|remote` 切换
3. **降级策略**: 链侧超时 → 标记该链不可用 → 通知运营 → 人工介入
