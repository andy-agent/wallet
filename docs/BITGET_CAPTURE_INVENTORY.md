# Bitget Capture Inventory

本文仅基于给定截图与 XML dump 编写，不依赖 adb、在线资料或仓库外上下文。

## 通用视觉基线

- 全部截图均为竖屏 `720 x 1600` PNG，整体为深色主题；主背景接近黑绿/深炭色，主 CTA 与高亮选中态为亮青蓝色。
- XML 根节点可见区域统一为 `720 x 1440`，说明截图额外包含状态栏与底部系统导航区；做像素级复刻时应以截图为最终像素真值，以 XML bounds 作为组件相对位置参考。
- 大部分一级页面都采用：顶部状态栏 + 居中标题或搜索框 + 中段功能卡片/内容区 + 底部圆角导航/操作条。
- 主要圆角风格：按钮与卡片普遍为大圆角；弹层为底部上拉 sheet，顶部有浅灰色拖拽条。

## 已知缺口与歧义

- `bitget_profile.png` 与 `bitget_profile.xml` 只能确认是带返回箭头的空白加载容器，无法从产物本身确定最终业务页面标题或加载中的骨架样式。
- `bitget_asset_detail.png` 视觉内容与 `bitget_asset_detail.xml` 存在明显不一致：截图中部是骨架占位和侧向残影，XML 却给出了完整业务文案；复刻时应以截图决定视觉层级，以 XML 决定文案与节点名称。
- `bitget_route_home.*` 更像行情/股票详情页而不是首页本身；由于未给出前一跳截图，只能将入口推断为 `首页/行情 -> 标的详情`。
- `bitget_discover.*` 所属一级信息架构不完全明确；从页面自身更接近交易工具/闪兑工具而非传统“发现页”。
- 全部截图为 `720 x 1600`，全部 XML 可见区域为 `720 x 1440`；像素级还原时需要额外补上状态栏与系统导航区高度差。
